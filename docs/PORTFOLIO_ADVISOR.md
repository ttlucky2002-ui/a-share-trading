# 持仓诊断与 DeepSeek 策略顾问

## 动机

用户已有自己的持仓，需要一个工具来：
1. 集中管理持仓记录（代码、数量、成本）
2. 自动获取实时行情，计算当前盈亏
3. 结合 DeepSeek 对今日市场热点的分析，给出逐只持仓的操作建议
4. 评估整体攻防态势

## 数据模型

```python
{
  "capital": 100000,          # 总资金
  "positions": [
    {
      "id": "p1",
      "code": "600519",
      "name": "贵州茅台",       # 录入时自动补全
      "quantity": 100,
      "cost_price": 1850.00,
      "added_at": "2026-05-28",
      "notes": ""
    }
  ]
}
```

运行时自动补充（不持久化）：
- `current_price` — 当前价
- `change_pct` — 今日涨跌幅
- `market_value` — 持仓市值
- `pnl` — 盈亏金额
- `pnl_pct` — 盈亏比例
- `weight` — 仓位占比

## 交互流程

```
用户打开"持仓诊断"页面
  → 页面加载已有持仓数据（GET /api/portfolio）
  → 显示持仓表格（代码/名称/数量/成本/现价/盈亏/仓位占比）
  → 用户可以增/删/改持仓
  → 点击"AI 策略分析"
  → 后端收集：持仓数据 + 市场概况 + 板块资金 + 热点概念 + 尾盘候选
  → 调用 DeepSeek
  → 返回逐只诊断 + 攻防策略
  → 前端渲染显示
```

## API 设计

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/api/portfolio` | 获取持仓（含实时行情） |
| POST | `/api/portfolio/save` | 保存持仓列表 |
| POST | `/api/portfolio/analyze` | 执行 DeepSeek 分析 |

## DeepSeek 提示词设计

发给 DeepSeek 的内容包含：
1. 市场概况（涨跌家数、涨停跌停、涨跌比）
2. 热门概念板块（前10名及其资金流入）
3. 行业板块资金排行（前10名）
4. 尾盘潜伏精选候选股（如有）
5. 用户持仓明细（代码、名称、数量、成本、现价、盈亏）

要求 DeepSeek 返回：
- 逐只持仓诊断：技术面、资金面、操作建议（增/持/减/清）
- 整体攻防评价：当前仓位风险、建议调整方向
- 热点匹配：持仓是否在当前热点上

## 前端页面
